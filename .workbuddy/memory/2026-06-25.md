# 2026-06-25 - AC-Nexus-OpenWRT Bug 修复

## 核心问题：CBI 设置保存后 config.json 不更新

### Bug #1 (CRITICAL) - CBI on_commit Lua 正则语法错误
**文件**: `acnexus/files/usr/lib/lua/luci/model/cbi/acnexus.lua:16`
**根因**: `[?%d*%]?` 被 Lua 解析为字符类（匹配 `?`, 数字, `*`, `%`, `]`），而非转义方括号。无法匹配 UCI 的 `acnexus.@acnexus[0].key=val` bracket 格式。
**修复**: 改为 `(%[%d+%])?`，正确使用 `%[` `%]` 转义。
**影响**: 整个 on_commit 的 UCI→config.json 同步逻辑静默失败。CBI 保存后 UCI 正确写入，但 config.json 永不被更新。

### Bug #2 - uci-defaults section reference 错误
**文件**: `acnexus/files/etc/uci-defaults/99-acnexus:7`
**根因**: 使用 `uci -q get acnexus.@settings[0]`，但 section type 是 `acnexus`，`settings` 只是名字。
**修复**: 改为 `@acnexus[0]`

### Bug #3 - weather_provider_set 被错误覆盖
**文件**: `acnexus/files/usr/lib/lua/luci/model/cbi/acnexus.lua:44`
**根因**: CBI 表单中没有 `weather_provider_set` 字段，但 `on_commit` 无条件将其设为 `false`。
**修复**: 仅在 UCI 明确存在该值时设置

### Bug #4 - daemon _sync_uci_to_config 遗漏字段
**文件**: `acnexus/files/usr/lib/acnexus/acnexus_service.py:189`
**根因**: 只同步了 4 个字段，遗漏 `api_key`、`qw_host`、`baidu_key`；位置变更不触发 save_config。
**修复**: 补全 3 个 API Key 字段同步，修复位置变更检测与保存逻辑
